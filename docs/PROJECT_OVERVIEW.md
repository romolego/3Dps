# Обзор проекта 3Dps

## Коротко

`3Dps` — это локальный инструмент для подготовки набора кадров из видео и последующей ручной разметки кадров в режиме быстрого просмотра.

Базовый сценарий выглядит так:

1. Загрузить видео или импортировать ZIP-проект.
2. Настроить диапазон, плотность и качество раскадровки.
3. Сгенерировать кадры.
4. Просматривать кадры как последовательность.
5. Ставить точечные метки и зоны.
6. При необходимости включать центрирование по ROI.
7. Экспортировать результат в архив для передачи или ревью.

## Зачем существует проект

Проект закрывает практическую задачу: быстро превратить видео в набор пригодных для ручного анализа кадров, не уходя в тяжёлые видеоредакторы и не поднимая отдельную серверную инфраструктуру.

Сильные стороны текущей модели:

- локальность и простота запуска;
- файловая переносимость проекта;
- быстрый переход от видео к разметке;
- возможность готовить материалы для передачи другим людям.

## Что уже подтверждено по текущей реализации

- локальный FastAPI-сервер на `127.0.0.1:8000`;
- SPA-фронтенд с экранами `upload`, `settings`, `viewer`;
- загрузка видео и импорт ZIP;
- хранение проектов в папке `projects/`;
- генерация кадров и миниатюр;
- точечные метки;
- зоны;
- типы меток и типы зон;
- экспорт в архив;
- расширенный экспорт;
- режим `read_only` для импортированных ZIP-проектов;
- глобальные настройки приложения;
- ROI и генерация центрированных кадров.

## Что важно понимать правильно

- Это не облачный сервис.
- Это не многопользовательская система с авторизацией.
- Это не полноценный NLE/видеоредактор.
- Это не 3D-движок, а инструмент псевдо-3D-навигации по подготовленным кадрам.

## Основные сущности

- `project` — единица хранения, привязанная к исходному видео или импортированному ZIP.
- `frame set` — набор сгенерированных кадров текущего проекта.
- `marker` — точечная аннотация на конкретном кадре.
- `zone` — прямоугольная область на конкретном кадре.
- `marker type` / `zone type` — пользовательский тип с цветом и параметрами отображения.
- `read_only project` — проект, созданный из импортированного ZIP, который можно просматривать и использовать для ревью, но не как обычный редактируемый мастер-проект.
- `centered frames` — отдельный набор кадров после трекинга ROI и сдвига изображения.

## Для кого этот проект

- для автора/оператора, который готовит кадры из видео;
- для человека, который размечает или проверяет кадры;
- для будущих ИИ-агентов, которым нужен понятный контекст перед изменениями системы.
