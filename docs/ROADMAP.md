# Roadmap и живой inbox проекта 3Dps

Базовая дата ревизии: `2026-03-14`.

## Вектор движения

Целевое направление проекта:

- сделать из `3Dps` устойчивый локальный инструмент для просмотра, разметки и передачи результатов по видеоматериалам;
- сохранить простоту запуска;
- наращивать ценность вокруг пометок, review-пакетов и скорости ручной работы.

## Что уже подтверждено в коде

- загрузка видео;
- импорт ZIP в `read_only`-режиме;
- генерация кадров;
- viewer;
- точечные метки;
- зоны;
- типы меток;
- типы зон;
- таймлайн-маркеры для кадров с пометками;
- карточки пометок текущего набора;
- расширенный экспорт;
- глобальные настройки приложения;
- ROI и центрированные кадры.

## Что выглядит частично закрытым относительно старого плана

- `Review-режим без инструментов`:
  частично закрыт через импорт ZIP в `read_only`, но это не отдельный законченный продуктовый режим.
- `Экспорт кадров с пометками`:
  частично закрыт через `export_advanced`, но формат финального review-пакета ещё можно развивать.
- `Горячие клавиши`:
  базовые клавиатурные сценарии есть, но отдельный завершённый слой навигации по меткам не подтверждён.

## Подтверждённый backlog из исторического плана

Следующие темы остаются актуальными и не выглядят полностью закрытыми:

- навигация `следующая/предыдущая метка`;
- навигация `следующий/предыдущий кадр с меткой`;
- отдельный кабинет со всеми метками по всем проектам;
- глобальный поиск по меткам и проектам;
- метка на диапазон кадров;
- review-пакет, который удобно открывать без сервера;
- экспорт/импорт профилей пресетов;
- сегменты внутри длинного проекта;
- кликабельные внутренние и внешние ссылки;
- обсуждения внутри метки.

## Рабочие приоритеты на ближайший цикл

### Приоритет A. Контур разметки как основа продукта

- довести UX меток и зон до стабильного предсказуемого состояния;
- добавить явную навигацию по пометкам;
- улучшить сценарии review без риска случайного редактирования.

### Приоритет B. Передача результата

- упаковать экспорт в понятный review-артефакт;
- стабилизировать состав экспортируемых метаданных;
- зафиксировать, какие форматы считаются внешним контрактом.

### Приоритет C. Скорость ручной работы

- развивать горячие клавиши;
- ускорять переходы между проектами, кадрами и пометками;
- добавить глобальный поиск и выборки.

### Приоритет D. Масштабирование сценария

- поддержка длинных видео и сегментов;
- переносимость пресетов между машинами;
- более удобный cross-project обзор.
