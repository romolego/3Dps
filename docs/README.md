# Набор системных документов 3Dps

Этот каталог нужен для быстрого входа в проект людям и ИИ-агентам, которые раньше его не видели.

Базовая дата аудита: `2026-03-14`.

## Состав

| Файл | Назначение | Когда обновлять |
| --- | --- | --- |
| `PROJECT_OVERVIEW.md` | Кратко объясняет, что это за продукт и зачем он нужен | Когда меняется смысл продукта или основной пользовательский сценарий |
| `ARCHITECTURE.md` | Описывает фактическую архитектуру, маршруты, API-группы и файловое хранение | Когда меняются компоненты, маршруты, хранение или инфраструктура |
| `AI_ROUTER.md` | Разводит роли между ChatGPT, system agent и dev-agent | Когда меняется рабочая схема взаимодействия агентов |
| `PROJECT_RULES.md` | Фиксирует ограничения, guardrails и правила изменения системы | Когда меняются продуктовые или технические инварианты |
| `ROADMAP.md` | Хранит целевое направление, backlog и живой inbox идей | Когда появляются новые идеи, приоритеты или статус реализации |
| `DECISIONS.md` | Журнал принятых решений с обоснованием | Когда принимается новое устойчивое решение |

## Рекомендуемый порядок чтения

1. `PROJECT_OVERVIEW.md`
2. `ARCHITECTURE.md`
3. `AI_ROUTER.md`
4. `PROJECT_RULES.md`
5. `ROADMAP.md`
6. `DECISIONS.md`

## Правило сопровождения

Если код изменил поведение системы, но документы не обновлены, документация считается устаревшей. Для этого проекта документация является частью системы, а не приложением к ней.
