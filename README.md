# Pseudo-3D from Video

Локальный веб-сервис для нарезки видео на кадры, интерактивного просмотра
(«крутилка»), рисования пометок поверх кадров и экспорта в ZIP.

## Требования

| Компонент    | Версия          | Как установить                                  |
|-------------|-----------------|------------------------------------------------|
| Python      | 3.10+           | https://python.org                              |
| FFmpeg      | любая актуальная | `winget install ffmpeg` или https://ffmpeg.org  |

> **Проверить FFmpeg:** откройте терминал и выполните `ffmpeg -version`.
> Если команда не найдена — скачайте архив с https://www.gyan.dev/ffmpeg/builds/
> (release full), распакуйте и добавьте папку `bin` в системную переменную PATH,
> **или** скопируйте `ffmpeg.exe` и `ffprobe.exe` в папку `bin/` внутри проекта.

## Быстрый старт (Windows)

1. Откройте **PowerShell** или **Terminal** в папке проекта (`D:\projects\3Dps`).

2. Установите Python-зависимости:
   ```powershell
   cd backend
   pip install -r requirements.txt
   ```

3. Запустите сервер:
   ```powershell
   python main.py
   ```
   Ожидаемый вывод:
   ```
     Open in browser: http://127.0.0.1:8000
   ```

4. Откройте в браузере: **http://127.0.0.1:8000**

## Как пользоваться

1. **Загрузка видео** — перетащите файл в область или нажмите для выбора.
2. **Настройки** — укажите начало / конец фрагмента (секунды или `мм:сс`),
   количество кадров N, ширину кадра и качество JPEG.
3. **Оценка** — нажмите «Estimate» чтобы увидеть приблизительный объём.
4. **Генерация** — нажмите «Generate Frames», дождитесь прогресс-бара.
5. **Просмотр** — листайте кадры:
   - стрелками клавиатуры ← →
   - перетаскиванием по изображению (влево-вправо)
   - кликом / перетаскиванием по таймлайну
   - ползунок «Drag sensitivity» регулирует скорость перемотки
   - ползунок «Zoom» масштабирует таймлайн
6. **Пометки** — нажмите «Draw» (или клавишу D), рисуйте карандашом,
   затем «Save». Пометки привязаны к кадрам и подгружаются автоматически.
7. **Экспорт** — кнопка «Export ZIP» скачивает архив со структурой:
   ```
   original_video/
   frames/
   annotations/    (PNG + JSON)
   index.json
   annotations_index.json
   ```

## Структура проекта

```
3Dps/
├── backend/
│   ├── main.py              # FastAPI-сервер
│   └── requirements.txt     # Python-зависимости
├── frontend/
│   └── index.html           # Единый HTML/CSS/JS-файл UI
├── projects/                # Хранилище проектов (создаётся автоматически)
│   └── <id>/
│       ├── original_*.mp4
│       ├── frames/
│       ├── thumbs/
│       ├── annotations/
│       ├── index.json
│       └── annotations_index.json
└── README.md
```

## Зависимости (Python)

- **fastapi** — веб-фреймворк
- **uvicorn** — ASGI-сервер
- **python-multipart** — обработка multipart/form-data (загрузка файлов)

Все остальные компоненты (FFmpeg, HTML Canvas, ZIP) работают
через стандартные средства без дополнительных библиотек.
