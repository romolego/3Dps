# 3Dps Agent Entry

Этот файл задаёт входную рамку для ИИ-агентов, которые работают с проектом `3Dps`.

## Что это за проект

`3Dps` — локальный веб-инструмент для разбора видео на кадры, просмотра кадров как псевдо-3D-последовательности, разметки кадров метками и зонами, а также экспорта результата в архивы для последующей передачи или ревью.

Проект сейчас ориентирован на:

- локальную работу на одной машине;
- FastAPI-бэкенд + SPA-фронтенд;
- файловое хранение проектов в папке `projects/`;
- ручное управление сервисом через `start_3dps.bat` и `stop_3dps.bat`.

## Маршрутизация ролей

- `ChatGPT` используется как помощник по формулированию идеи, требований и ТЗ.
- `System agent` используется как ревизор системы: актуализирует документы, правила, backlog и журнал решений.
- `Cloud Code`/локальный dev-agent используется как исполнитель продуктовой разработки: код, багфиксы, рефакторинг, тестирование.

Если запрос про реализацию функционала, первичный исполнитель — dev-agent.
Если запрос про правила, рамки, контекст, ревизию системы и фиксацию договорённостей, первичный исполнитель — system agent.

## Обязательные инварианты

- Не считать проект облачным сервисом: это локальный инструмент.
- Не ломать совместимость файловой структуры `projects/<id>/` без явной миграции.
- Не менять без причины канонический URL проекта `/project/<id>`.
- Не убирать семантику `read_only` у импортированных ZIP-проектов без отдельного решения.
- Не считать временные `agent log`, `FIX H*`, `.cursor/debug.log` и обращения к `127.0.0.1:7248` частью целевой архитектуры.

## Обязательное сопровождение изменений

Если меняется архитектура, поведение продукта, хранилище, маршрутизация, экспортный формат или рабочие правила, нужно синхронно обновить:

- [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)
- [docs/PROJECT_RULES.md](docs/PROJECT_RULES.md)
- [docs/ROADMAP.md](docs/ROADMAP.md)

## Если задача неочевидна

- Сначала зафиксировать фактологию по текущему состоянию проекта.
- Затем отделить подтверждённую реализацию от идеи или гипотезы.
- После этого либо обновить системные документы, либо передать в разработку с точным техническим заданием.
